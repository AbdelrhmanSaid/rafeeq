# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Report vulnerabilities privately through GitHub Security Advisories for this repository. Include:

- A clear description of the vulnerability.
- Steps to reproduce or proof-of-concept details.
- The affected area, browser, device, or deployment context.
- Any suggested mitigation, if known.

The maintainers will review the report and follow up as soon as practical.

## Supported Versions

Security fixes are handled on the default branch. If a released version is affected, the fix will be included in the next available release or deployment.
